The `sinc` CLI is the main entrypoint.

## Usage

```bash
sinc [--setup] [--list] [--kill <id>] [--uninstall] [--resume] [--quiet] [--verbose] [--json]
```

## Commands

- `sinc` - Connect to the VPS from the current project directory
- `sinc --setup` - interactive setup (writes config + checks dependencies)
- `sinc --list` - list active sessions
- `sinc --kill <id>` - kill a session
- `sinc --uninstall` - remove local config and sync sessions

## Flags

- `-h, --help` - show help
- `-V, --version` - show version
- `-r, --resume` - resume an existing session
- `-q, --quiet` - suppress non-essential output
- `-v, --verbose` - show diagnostic output
- `--json` - emit machine-readable output
- `--completions <shell>` - print shell completions (`bash`, `zsh`, `fish`)

## Output modes

- `--quiet` suppresses non-essential output
- `--verbose` includes diagnostic output
- `--json` emits machine-readable events (useful for wrappers)

## Environment variables

- `SINC_NO_UPDATE_CHECK=1` disables GitHub release checks

## Exit codes

- `0` success
- `1` general error
- `2` misuse (bad flags, trying to resume when no session exists, etc.)
- `69` dependency unavailable
- `76` connect error
- `78` config error

## Examples

```bash
# first-time setup
sinc --setup

# connect
sinc

# list sessions
sinc --list

# kill a session
sinc --kill abc123
```
