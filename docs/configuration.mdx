title: Configuration

---

This file controls how `sinc` connects to your VPS and how syncing behaves.

## Location

The CLI reads a single global config file:

- `~/.config/sincronizado/config.json`

The setup wizard (`sinc --setup`) creates/updates it for you.

## Schema

```json
{
  "vps": {
    "hostname": "string",
    "user": "ubuntu",
    "port": 22
  },
  "sync": {
    "mode": "both",
    "ignore": ["node_modules", ".venv", ".git"],
    "remoteBase": "~/workspace"
  },
  "agent": "opencode",
  "ssh": {
    "connectTimeout": 10,
    "keepaliveInterval": 60,
    "identityFile": "~/.ssh/id_ed25519"
  },
  "connection": {
    "protocols": ["ssh", "et", "mosh"],
    "reconnect": {
      "maxAttempts": 5,
      "baseDelayMs": 1000,
      "maxDelayMs": 10000
    }
  }
}
```

## Options

### vps.hostname

- required
- the host name or IP for SSH (tailscale hostname recommended)

### vps.user

- default: `ubuntu`

### vps.port

- default: `22`

### sync.mode

- one of: `none`, `pull`, `push`, `both`
- default: `both`

### sync.ignore

- list of ignore patterns (applied in addition to any `.syncignore` in the project)

### sync.remoteBase

- base directory on the VPS to place projects
- default: `~/workspace`

### agent

- one of: `opencode`, `claude`
- default: `opencode`

### ssh.identityFile

- optional
- path to a private key (if you want to force a specific key)

### connection.protocols

- ordered list of protocols to attempt
- default: `ssh`, `et`, `mosh`

## Example

```json
{
  "vps": {
    "hostname": "oracle.tail1234.ts.net",
    "user": "ubuntu",
    "port": 22
  },
  "sync": {
    "mode": "both",
    "ignore": ["node_modules", ".venv", ".git"],
    "remoteBase": "~/workspace"
  },
  "agent": "opencode"
}
```

## Troubleshooting

### Config not loading

1. Validate JSON: `python -m json.tool < ~/.config/sincronizado/config.json`
2. Ensure the file is readable

### VPS connection failing

1. Verify SSH works: `ssh user@hostname "echo ok"`
2. If you use Tailscale, confirm both sides are connected: `tailscale status`
